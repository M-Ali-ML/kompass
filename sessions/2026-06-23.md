## ⚠️ Gotchas & Troubleshooting
- **Gemini API 503 Errors**:
  - *Issue*: Experienced connection outages / "Service Unavailable" errors when querying the Gemini 2.5 flash models.
  - *Resolution*: Investigated env parameters and regional keys. Transitioned to a more robust, local fallback/mock structure when `LLM_MODEL=test` is active.
- **ExceptionGroup in Python MCP SDK**:
  - *Issue*: Pydantic model `CallToolResult` raised an AttributeError for `is_error` because it is named `isError` (camelCase) in the python MCP SDK. It threw an ExceptionGroup, crashing the backend.
  - *Resolution*: Changed `result.is_error` to `getattr(result, "isError", False)`. Added explicit `BaseExceptionGroup` flattening logic in FastAPI routes to detect and format sub-exceptions.
- **Next.js Dev Error Overlay**:
  - *Issue*: Catching backend 503 errors and throwing a new `Error` in async event handlers caused Next.js to show the red dev overlay.
  - *Resolution*: Appended error messages directly to the React state log instead of throwing `Error`, preventing the dev overlay.

## 🧠 Preferences & Architectural Decisions
- **Unified SSE Client-Server Adapter**:
  - *Decision*: Adopted `AGUIAdapter` SSE stream interface directly, ensuring a clean, standard communication protocol between Pydantic AI backend and CopilotKit frontend components, without custom route serialization logic.
- **Auto-Collapsing Reasoning Steps**:
  - *Decision*: Implemented auto-collapse on completed thinking states to avoid cluttering the UI, while keeping the steps inspectable manually for transparency.
- **Local E2E Fallback Model**:
  - *Decision*: Setting `LLM_MODEL=test` skips calling the Gemini API and routes calculations directly to the `MCPMathServiceAdapter`. This allows fast, offline, and reliable E2E tests.
- **Stateless Backend**:
  - *Decision*: Kept backend completely stateless to minimize session tracking complexity. Chat history is managed by the client.
