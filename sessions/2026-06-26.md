## ‚öÝÔ∏è Gotchas & Troubleshooting
- **"‚âà approx" with `MCP_MODE=live`** ‚Äî two distinct causes uncovered:
  1. **Stale process**: `uvicorn --reload` only watches `.py` files, NOT `.env`, and `config.py` evaluates `Settings()` once at import. Flipping `.env` mock‚Üílive has no effect until a **full backend restart**. (Shell `$MCP_MODE` was empty, ruling out an env override.)
  2. **`pydantic-settings` precedence**: a real exported `MCP_MODE` env var overrides the `.env` value (verified: `MCP_MODE=mock uv run ...` ‚Üí `mock` even with `.env=live`).
  - After restart, confirmed LIVE via [mcp_calls.jsonl](file:///Users/aly/repos/kompass/backend/logs/mcp_calls.jsonl) (`"mock": false`, real Jamaica fares ‚Ç¨807‚Äì848, real hotels). Tell-tale of mock: "Aegean Airlines" BER‚Üíanywhere at flat ‚Ç¨120.
- **Jamaica showed approx (live)**: `search_flights BER‚ÜíMBJ max_stops=0` ‚Üí `available:false` because **no nonstop BER‚ÜíJamaica exists** (user has `direct_flights_only=True`). Agent fabricated an impossible "BER 08:00 ‚Üí MBJ 10:00" leg and flagged the scenario approx ‚Üí addressed by the direct-only prompt fix above.
- **Running tests**: use `cd backend && uv run python -m pytest -q` (plain `uv run pytest` fails with `ModuleNotFoundError: app` ‚Äî the `python -m` form puts cwd on `sys.path`; `python` alone isn't on PATH).
- **Lint warnings** on `pydantic`/`fastapi`/`pydantic_ai` imports are pre-existing (basedpyright can't resolve the venv) ‚Äî not real errors.
