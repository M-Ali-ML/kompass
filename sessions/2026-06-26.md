## ⚠️ Gotchas & Troubleshooting
- **"≈ approx" with `MCP_MODE=live`** — two distinct causes uncovered:
  1. **Stale process**: `uvicorn --reload` only watches `.py` files, NOT `.env`, and `config.py` evaluates `Settings()` once at import. Flipping `.env` mock→live has no effect until a **full backend restart**. (Shell `$MCP_MODE` was empty, ruling out an env override.)
  2. **`pydantic-settings` precedence**: a real exported `MCP_MODE` env var overrides the `.env` value (verified: `MCP_MODE=mock uv run ...` → `mock` even with `.env=live`).
  - After restart, confirmed LIVE via [mcp_calls.jsonl](file:///Users/aly/repos/kompass/backend/logs/mcp_calls.jsonl) (`"mock": false`, real Jamaica fares €807–848, real hotels). Tell-tale of mock: "Aegean Airlines" BER→anywhere at flat €120.
- **Jamaica showed approx (live)**: `search_flights BER→MBJ max_stops=0` → `available:false` because **no nonstop BER→Jamaica exists** (user has `direct_flights_only=True`). Agent fabricated an impossible "BER 08:00 → MBJ 10:00" leg and flagged the scenario approx → addressed by the direct-only prompt fix above.
- **Running tests**: use `cd backend && uv run python -m pytest -q` (plain `uv run pytest` fails with `ModuleNotFoundError: app` — the `python -m` form puts cwd on `sys.path`; `python` alone isn't on PATH).
- **Lint warnings** on `pydantic`/`fastapi`/`pydantic_ai` imports are pre-existing (basedpyright can't resolve the venv) — not real errors.
- **Stray root `package.json`/`package-lock.json`/`node_modules`**: running `npm install --save` (outside the sandbox) created a **repo-root** package set, which made Next.js pick the repo root as workspace root → `Cannot find module '@copilotkit/react-core/v2/styles.css'`. Fix: deleted the three stray root artifacts and pinned `turbopack.root` to the frontend dir in [next.config.mjs](file:///Users/aly/repos/kompass/frontend/next.config.mjs). Always install deps with `working_directory: frontend` and verify no root `package.json` appears afterward.
- **`@vis.gl/react-google-maps` library split**: `Polyline` is in the **`maps`** library but `LatLngBounds` is in **`core`** → `mapsLib.LatLngBounds is not a constructor`. Fix: also `useMapsLibrary("core")` and build bounds from it.
- **`react-resizable-panels` v4 API change**: it is NOT the classic `PanelGroup`/`PanelResizeHandle`/`direction` API. v4 exports `Group`/`Panel`/`Separator` with `orientation`, `defaultLayout`/`onLayoutChanged` (no `autoSaveId`), `groupRef` (accepts a plain `useRef`; `getLayout()/setLayout()` use a `{panelId: percent}` map). **Numeric size props = PIXELS**; percentages must be strings (`"18%"`) — passing `defaultSize={18} maxSize={32}` locked the sidebar to 12–32 **px** (the "stuck, can't widen" bug).
- **Google Maps `InvalidKeyMapError`**: env wiring is correct; this is a GCP-side issue. Needs Maps JavaScript API + Geocoding API enabled, **billing enabled**, and key referrer restrictions allowing `http://localhost:3000/*` (or None for dev). The key in `frontend/.env` looked 40 chars (normal is 39) — worth re-copying.

## 🧠 Preferences & Architectural Decisions
- **Phase 7 = web search, not a transit MCP**: chosen for ferry coverage + zero new infra; times/prices framed as approximate. Google Routes API noted as a future "exact rail/bus times" enhancement only.
- **Prompts live as `.md` behind `PromptServicePort`**: both `system_prompt` and `research_prompt` resolve through `FilePromptService`; a future `LangfusePromptService` is a single swap. (Note: research prompt is read once at import — a Langfuse version would fetch per-run/cached.)
- **Logging is infra, co-located with telemetry**: `configure_logging()` in `logging_config.py`, not in the entrypoint. `LOG_LEVEL`/`MCP_MODE`/prompt are read at startup → require a restart to change.
- **`SecretStr` over a custom masker**: idiomatic auto-masking; trade-off is the fixed `**********` (lost the old `7008…cea7 (len=64)` hint — re-add a `field_serializer` if that detail is ever wanted).
- **Map = frontend-only, scenario-driven**: per the confirmed coords decision, geocoding happens client-side (Google Geocoder, cached) — no `Leg`/`Accommodation` schema changes, no new backend MCP. The map reflects the agent's `generate_scenarios` output via a shared client store rather than CopilotKit `useCopilotReadable` (sharing the viewport *to* the agent was deemed low-value).
- **Resizable layout pinned to percentages + versioned storage**: panel sizes are `%` strings; `localStorage` keys carry a `:v2` suffix so the earlier px-bug layouts are discarded. Map panel gated at `lg` (1024px) and conditionally rendered.
