## ⚠️ Gotchas & Troubleshooting
- **Playwright browser arch**: under the Cursor sandbox the test worker resolves a `mac-x64` browser path even though Node is arm64, so browser specs fail with "Executable doesn't exist". Running `npm run test:e2e` outside the sandbox (full permissions) detects arm64 correctly and all specs pass. `npx playwright install chromium` was needed once.
- **CopilotKit dev inspector**: `<cpk-web-inspector>` renders a viewport overlay that intercepts clicks on the header. The settings e2e injects `cpk-web-inspector{display:none !important}` via `addStyleTag` before clicking the gear. (Note: this overlay appears even in the production e2e build — likely from `agents__unsafe_dev_only` in [copilotkit-provider.js](file:///Users/aly/repos/kompass/frontend/app/copilotkit-provider.js); worth revisiting.)
- **eslint noise from build dirs**: `npm run lint` picked up the `.next-e2e` Playwright build output; added `.next-e2e/**` to `globalIgnores` in [eslint.config.mjs](file:///Users/aly/repos/kompass/frontend/app/../eslint.config.mjs).
- **set-state-in-effect**: dropped an effect in `page.js` that closed the drawer on resize (the drawer isn't rendered on desktop anyway), avoiding the repo's enforced `react-hooks/set-state-in-effect` rule.

## 🧠 Preferences & Architectural Decisions
- **HITL over backend tool**: the clarifying-question feature is a *frontend* `useHumanInTheLoop` tool; PydanticAI's AG-UI `_AGUIFrontendToolset` exposes it to the agent and handles the deferred result, so no backend handler/port was added.
- **Manual overrides / iterative refinement are prompt-only** (no new domain/constraint schema), per the user's choice. `comparison_label` is the stable scenario handle for refinement.
- **Vitest** chosen for frontend unit tests (Node env, `tests/unit/**`), kept separate from Playwright (`tests/e2e/**`) so the runners never collide.
