## ⚠️ Gotchas & Troubleshooting
- **No Working Agent**: Attempting to implement a fully-featured travel application via pure "vibe coding" resulted in an unstable system with broken agent integration.
- **Lesson Learned**: Start with a very simple, working baseline and progress incrementally. Speculative scaffolding must be avoided.
- **LLM Loop Prevention & Limits**: The LLM agent was entering infinite tool calls loops due to generic/unsatisfying mock responses in local test adapters. Enforcing `UsageLimits` inside the agent runner and improving mock search results for destinations cleanly intercepts potential loops.

## 🧠 Preferences & Architectural Decisions
- **Start Very Simple**: Build the absolute minimal slice of functionality first, then expand it.
- **Test-Driven Development (TDD)**: Do not vibe code. Strictly implement TDD cycles as defined in [tdd.md](file:///Users/aly/repos/kompass/.agents/rules/tdd.md) by writing tests first and keeping implementation code minimal.
- **Separation of Routing & Orchestration**: Routing handlers should only parse HTTP request envelopes. Orchestration/business flows must remain in application services.
